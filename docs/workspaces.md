# Workspaces

Um **workspace** é um projeto independente — como uma pasta raiz para suas APIs.

## Para que serve

- Separar projetos diferentes (ex.: "API Cliente A" vs "API Interna")
- Cada workspace tem suas próprias coleções, ambientes e workflows
- O workspace ativo aparece no **header**, no seletor ao lado do nome Rebeka

## Criar workspace

1. Clique no seletor de workspace no header.
2. Use a opção de **novo workspace** (ou equivalente no menu).
3. Dê um nome descritivo.

## Trocar de workspace

1. Selecione outro workspace no header.
2. A sidebar e o painel central carregam os dados daquele projeto.

> Ao trocar, o app salva o estado atual antes de carregar o novo workspace.

## Excluir workspace

Use a ação de excluir na sidebar (ícone de lixeira no topo da área de coleções). **Esta ação remove o workspace e todo o conteúdo associado** — faça backup com **Exportar .bek** antes, se necessário.
